# BaseClient - Multi-Service SaaS Client Application

A cross-platform React Native + Expo client application for multi-tenant SaaS platforms. Built to work with the microservices architecture including IdentityService, OnlineMenuService, and QuestionerService.

- [API Hooks Guide](docs/API_HOOKS_GUIDE.md)
- [Code Standards](docs/react-code-standards.md)
- [Packages Development Guide](packages/PACKAGES_GUIDE.md) - Build, debug, and release local packages

## Architecture Overview

This client application connects to the following backend services:

### Backend Services

1. **IdentityService** (Port 5002)
   - Authentication and authorization
   - User management (CRUD operations)
   - Tenant management and configuration
   - OTP authentication (SMS/Email)
   - Keycloak integration

2. **OnlineMenuService** (Port 5006)
   - Business logic for tenant menus
   - Multi-tenant data isolation

3. **QuestionerService** (Port 5004)
   - Questionnaire templates management
   - Completed questionnaires storage
   - Multi-tenant data isolation

4. **ContentService** (Port 5009)
   - File uploads and storage
   - Image processing
   - SeaweedFS integration

5. **NotificationService** (Port 5015)
   - SMS notifications
   - Email notifications
   - Notification templates

## Getting Started

### 1. Install Dependencies

```bash
cd BaseClient
npm install
```

### 2. Build Local Packages

The app uses local packages that need to be built:

```bash
# Build identity-client package
cd packages/identity-client && npm install && npm run build && cd ../..
```

### 3. Start the Application

```powershell
# Development (default)
npm run start:dev

# Test environment
npm run start:test

# Production environment
npm run start:prod
```

The app will start on **http://localhost:8082** by default.

### 4. Run on Specific Platforms

```bash
# Web
npm run web

# Android (requires Android SDK)
npm run android

# iOS (requires macOS with Xcode)
npm run ios
```

## API Hooks Generation

The app uses [Orval](https://orval.dev/) to auto-generate React Query hooks from OpenAPI specs.

### Generate Hooks

```powershell
# Generate hooks from running services (downloads swagger specs)
npm run generate:hooks

# Generate using existing swagger files (skip download)
npm run generate:hooks:skip-download
```

### Manual Generation

```powershell
# With custom URLs
powershell ./scripts/Generate-ApiHooks.ps1 `
  -OnlineMenuUrl "https://localhost:5006" `
  -IdentityUrl "http://localhost:5002" `
  -QuestionerUrl "https://localhost:5004" `
  -ContentUrl "http://localhost:5009" `
  -NotificationUrl "http://localhost:5015"
```

For a complete guide on adding new services, see [API Hooks Guide](docs/API_HOOKS_GUIDE.md#-adding-a-new-service-step-by-step-guide).

### Using Generated Hooks

```typescript
// Identity APIs
import { useLogin, useListUsers } from '@/server/autoGeneratedHooks/identity/auth';
import { useListTenants } from '@/server/autoGeneratedHooks/identity/tenants';

// Business APIs
import { useListTenantMenus } from '@/server/autoGeneratedHooks/onlinemenu/tenantmenus';
import { useListQuestionerTemplate } from '@/server/autoGeneratedHooks/questioner/questionertemplates';

// Content APIs
import { useRequestUploadUrl, useCompleteUpload } from '@/server/autoGeneratedHooks/content';

// Notification APIs
import { useSendSms, useSendEmail } from '@/server/autoGeneratedHooks/notification';

function MyComponent() {
  const { mutate: login } = useLogin();
  const { data: users } = useListUsers();
  const { data: templates } = useListQuestionerTemplate();

  return (/* ... */);
}
```

## Authentication Flow

1. User authenticates via IdentityService (`/api/auth/login`)
2. IdentityService validates credentials with Keycloak
3. JWT token returned with claims (tenantId, roles)
4. Token stored securely and included in API requests
5. All API calls automatically filtered by tenant

## Troubleshooting

### "Network Error" on API calls

- Verify backend services are running
- Check environment configuration URLs
- Ensure CORS is configured on backend services

### "Unable to resolve module"

- Run `npm install` in the root directory
- Build local packages in `packages/` directory
- Clear Metro cache: `npm start -- --clear`

### Expo update warnings

```bash
# Fix expo package versions
npx expo install --fix
```
