## Problem Statement

After login, the frontend needs to fetch the current tenant's theme configuration from the API and inject it into the ThemeProvider. This should be cached locally to prevent flash-of-default-theme on subsequent page loads.

---

## Requirements

### Theme Fetch Flow

1. **On app load** (before login):
   - Check localStorage for cached theme config + tenantId
   - If found and not expired, apply immediately (prevents flash)
   - If not found, use DEFAULT_THEME_CONFIG

2. **On login success**:
   - Extract `tenantId` from user info (already in Redux `userInfo.tenantId`)
   - Call `GET /api/tenants/{tenantId}/theme`
   - Store response in ThemeProvider via `setTenantConfig()`
   - Cache in localStorage with timestamp

3. **On subsequent page loads** (already logged in):
   - Load cached theme immediately from localStorage
   - Background-fetch fresh theme with ETag (If-None-Match)
   - If 304 (unchanged): keep cached
   - If 200 (changed): update ThemeProvider + cache

4. **On logout**:
   - Clear cached theme
   - Revert to DEFAULT_THEME_CONFIG

### React Query Integration
Use the existing orval-generated hooks pattern:
- Create a `useTenantTheme()` hook that wraps the API call
- Use React Query's `staleTime` and `cacheTime` for cache management
- Enable background refetching with ETag support

### Logo URL Resolution
When the theme config includes `logoContentId`:
- Call ContentService's `GET /content/{logoContentId}/public-url` to resolve the actual URL
- Cache the resolved URL alongside the theme config
- Handle missing/deleted content gracefully (fallback to default logo)

### Error Handling
- API unreachable: Use cached theme, fall back to defaults
- Invalid theme response: Fall back to defaults, log warning
- Tenant has no theme configured: API returns defaults, frontend uses as-is

## Cache Key Structure

```
localStorage key: 'tenant-theme-{tenantId}'
value: JSON.stringify({
  config: TenantThemeConfig,
  logoUrl: string | null,
  faviconUrl: string | null,
  etag: string,
  cachedAt: number (timestamp)
})
```

Max cache age: 24 hours (force refetch after that even without ETag support).
