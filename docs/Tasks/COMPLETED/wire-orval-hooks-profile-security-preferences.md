# Wire Orval API Hooks into Profile/Security/Preferences Screens

## Status: COMPLETED

## Problem Statement
The Orval hooks have been generated at `src/server/autoGeneratedHooks/identity/me/me.ts` but the UI screens still use placeholder data from `useAuth()` or just log to console. We need to wire the generated hooks into the four screen components.

## Changes Made

### 1. ProfileSettingsScreen.tsx
- Replaced `useAuth()` with `useIdentityServiceAPIEndpointsMeGetProfile()` query
- Added `useIdentityServiceAPIEndpointsMeUpdateProfile()` mutation for save
- Added loading spinner (ActivityIndicator) and error state
- Initialize form fields from query data via `useEffect`
- Invalidate profile query on successful mutation via `.catch(() => {})` pattern
- Extracted AccountInfoSection to a separate component to stay under 200-line limit
- Show "Saving..." label on button while mutation is pending

### 2. AccountInfoSection.tsx (NEW)
- Extracted from ProfileSettingsScreen to keep it under 200 lines
- Displays read-only role and tenant badges

### 3. ChangePasswordForm.tsx
- Wired `useIdentityServiceAPIEndpointsMeChangePassword()` mutation
- On success: clear form, show success notification via `notifySuccess()`
- On error: show error notification via `notifyError()`
- Disable submit button while mutation is pending
- Show "Saving..." label on button while pending

### 4. SecuritySettingsScreen.tsx
- Wired `useIdentityServiceAPIEndpointsMeListSessions()` query for loading sessions
- Wired `useIdentityServiceAPIEndpointsMeRevokeSession()` mutation for revoking
- Display actual session list with loading/error/empty states
- Invalidate sessions query on successful revoke

### 5. SessionItem.tsx (NEW)
- Extracted session row rendering into a dedicated component
- Displays IP address, last access time, client info, and revoke button
- Uses FM() with `{{p1}}` placeholders for session data display

### 6. PreferencesSettingsScreen.tsx
- Wired `useIdentityServiceAPIEndpointsMeGetPreferences()` query
- Wired `useIdentityServiceAPIEndpointsMeUpdatePreferences()` mutation
- Initialize dropdowns from query data via `useEffect`
- Added loading spinner and error state
- Invalidate preferences query on successful mutation

### 7. Translation keys (en.json)
- Added `settings.security.sessionIp`, `sessionLastAccess`, `sessionClient`
- Added `settings.security.messages.sessionRevokeError`

### 8. TestIds (profileTestIds.ts)
- Added `SECURITY_SETTINGS_LOADING`, `SECURITY_SETTINGS_ERROR`, `SECURITY_SESSION_ITEM`
- Added `PREFERENCES_SETTINGS_LOADING`, `PREFERENCES_SETTINGS_ERROR`
