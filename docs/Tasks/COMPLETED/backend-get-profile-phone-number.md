# Backend: Add phoneNumber to GET /me/profile Response

## Problem Statement
The `GET /me/profile` endpoint returns `UserProfileResponse` which does not include `phoneNumber`. The `PUT /me/profile` endpoint accepts `phoneNumber`, so users can save a phone number but it disappears on page reload because the GET response never returns it.

## Root Cause
- `UserListItem` (from Identity.Abstractions NuGet package) lacks a `PhoneNumber` property
- `MapToUserListItem` in Identity.Keycloak does not extract `phoneNumber` from Keycloak user attributes
- `UserProfileResponse` record in GetProfile.cs does not include `phoneNumber`

## Approach (Option B - Smallest Blast Radius)
1. Add `PhoneNumber` property to `UserListItem` in Identity.Abstractions
2. Update `MapToUserListItem` in Identity.Keycloak to extract phone from Keycloak attributes
3. Bump NuGet package versions (patch)
4. Build local packages and update IdentityService references
5. Add `PhoneNumber` to `UserProfileResponse` and mapping in GetProfile.cs
6. Rebuild identity API and regenerate Orval hooks
