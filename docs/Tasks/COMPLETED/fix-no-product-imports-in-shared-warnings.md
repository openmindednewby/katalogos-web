# Fix no-product-imports-in-shared ESLint Warnings

## Problem
23 `no-product-imports-in-shared` ESLint warnings existed because product-specific components lived in `src/components/` (shared layer) but imported from product-specific modules (autoGeneratedHooks for onlinemenu/questioner/notification, features/, modules/).

## Solution

### Phase 1: Move product-coupled files
Moved files with direct product-specific imports OUT of `src/components/` into `src/features/{domain}/`:

1. **QuestionerTemplates** (13 warnings) - Moved entire `src/components/QuestionerTemplates/` to `src/features/questioner/components/`
2. **OnlineMenus** (2 warnings) - Moved `FullMenuEditor.tsx` and `MenuEditorModal.tsx` to `src/features/onlinemenus/components/`
3. **Dashboard hooks** (3 warnings) - Moved `useDashboardData.ts` and `useWelcomeWizard.ts` to `src/features/dashboard/hooks/`
4. **Settings notification** (2 warnings) - Moved `DisplayPreferenceDropdown.tsx` and `notificationPreferencesHelpers.ts` to `src/features/notification/`

### Phase 2: Fix transitive violations
Moving hooks/utils to `src/features/` caused their consumers in `src/components/` to import from `@/features/`, which the ESLint rule also flags. Fixed by moving consumer components too:

5. **Dashboard components** (3 warnings) - Moved `DashboardPage.tsx`, `WelcomeWizard.tsx`, `WizardStepContent.tsx` to `src/features/dashboard/components/`
6. **Notification Settings components** (4 warnings) - Moved `CategoryPreferenceRow.tsx`, `NotificationCategoriesSection.tsx`, `NotificationPreferencesScreen.tsx` to `src/features/notification/components/`

### Key decisions
- Exported `WelcomeWizardState` interface from the hook so feature components can import the type directly
- Dashboard barrel (`src/components/Dashboard/index.ts`) emptied; route imports directly from features
- Settings barrel updated to remove notification-related re-exports; route imports directly from features
- Test files co-located with their source files in the new locations
- Shared components (BusinessProfileNudge, GuidedActionCard, OverviewCard, etc.) remain in `src/components/` since they have no product-specific imports
