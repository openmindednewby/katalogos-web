# Fix Privacy Settings Code Review Issues

## Problem Statement
6 code review issues were identified in the GDPR Privacy Settings frontend code that need to be addressed.

## Issues

### Issue 1 (MEDIUM) - Manual API hooks instead of generated Orval hooks
- Files: `usePrivacyConsent.ts`, `useDataExport.ts`, `useAccountDeletion.ts`
- These hooks use `identityInstance` directly. Generated Orval hooks exist in `autoGeneratedHooks/identity/privacy/privacy.ts`.
- Action: Refactor wrapper hooks to use generated hooks instead of direct `identityInstance` calls.

### Issue 2 (MEDIUM) - Weak typing in types.ts
- `ConsentRecord.consentType` and `UpdateConsentRequest.consentType` typed as `string` instead of `ConsentType`.
- Action: Change both fields to `consentType: ConsentType`.

### Issue 3 (MEDIUM) - Magic numbers in styles
- Files: `DeletionRequestForm.tsx`, `ConsentManagement.tsx`, `DataExportSection.tsx`, `AccountDeletionSection.tsx`
- Action: Create `constants.ts` with named style constants, refactor all components.

### Issue 4 (MEDIUM) - Double blank line in AccountDeletionSection.tsx
- Around line 9-10 between import groups.
- Action: Remove extra blank line.

### Issue 5 (MEDIUM) - Inconsistent callback pattern in DataExportSection.tsx
- `downloadErrorCallbacks` uses anonymous function inside `useMemo`.
- Action: Extract to named `useCallback` handler (`handleDownloadError`).

### Issue 6 (LOW) - Magic number `marginTop: 8` in PrivacySettingsScreen.tsx
- Action: Extract to named constant `ERROR_TEXT_MARGIN_TOP`.
