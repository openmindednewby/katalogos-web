# API Hooks Generation Guide

This guide explains how to generate React Query hooks for both OnlineMenuSaaS and IdentityService APIs.

## 🎯 Overview

The app connects to **five microservices**:
- **IdentityService** (Port 5002) - Authentication, users, tenants
- **QuestionerService** (Port 5004) - Questionnaire templates and answers
- **OnlineMenuSaaS** (Port 5006) - Business logic (menus, categories)
- **ContentService** (Port 5009) - File uploads and storage
- **NotificationService** (Port 5015) - SMS and email notifications

API hooks are auto-generated using [Orval](https://orval.dev/) from OpenAPI/Swagger specs.

## 🚀 Quick Start

### 1. Start Both Services

```powershell
# Option A: Use the unified startup script (easiest)
cd C:\desktopContents\projects\OnlineMenuSaaS
.\Start-AllServices.ps1 -Docker

# Option B: Start individually
cd C:\desktopContents\projects\IdentityService
docker-compose up -d

cd C:\desktopContents\projects\OnlineMenuSaaS
docker-compose up -d
```

### 2. Generate Hooks

```powershell
cd clients\OnlineMenuClientApp

# Generate hooks from running services
npm run generate:hooks

# Or manually with custom URLs
powershell ./scripts/Generate-ApiHooks.ps1 -OnlineMenuUrl "https://localhost:5006" -IdentityUrl "http://localhost:5002"
```

### 3. Use the Hooks

```typescript
// Import hooks from the appropriate service
import { useListTenantMenus } from '@/server/autoGeneratedHooks/onlinemenu';
import { useLogin, useListUsers } from '@/server/autoGeneratedHooks/identity';

function MyComponent() {
  // OnlineMenuSaaS API
  const { data: menus } = useListTenantMenus();

  // IdentityService API
  const { mutate: login } = useLogin();
  const { data: users } = useListUsers();

  return (
    // Your component code
  );
}
```

## 🔧 Configuration

### Orval Configuration ([orval.config.js](orval.config.js))

```javascript
module.exports = defineConfig({
  // OnlineMenuSaaS Business API
  onlineMenuApi: {
    input: {
      target: './src/server/swagger/onlinemenu.json',
    },
    output: {
      target: './src/server/autoGeneratedHooks/onlinemenu/index.ts',
      client: 'react-query',
      mode: 'tags-split',
      override: {
        mutator: {
          path: './src/server/httpClient.ts',
          name: 'customInstance',
        },
      },
    },
  },

  // IdentityService API
  identityApi: {
    input: {
      target: './src/server/swagger/identity.json',
    },
    output: {
      target: './src/server/autoGeneratedHooks/identity/index.ts',
      client: 'react-query',
      mode: 'tags-split',
      override: {
        mutator: {
          path: './src/server/httpClientIdentity.ts',
          name: 'identityInstance',
        },
      },
    },
  },
});
```

### HTTP Client Configuration

**For OnlineMenuSaaS** ([httpClient.ts](src/server/httpClient.ts)):
- Uses `API_URL` from environment (default: https://localhost:5006)
- Handles business logic APIs

**For IdentityService** ([httpClientIdentity.ts](src/server/httpClientIdentity.ts)):
- Uses `IDENTITY_API_URL` from environment (default: http://localhost:5002)
- Handles auth, users, tenants

If you prefer to generate manually:

```powershell

Invoke-RestMethod -Uri "https://localhost:5006/swagger/v1/swagger.json" `
```


## 🔍 Troubleshooting

### "Failed to download spec"

**Problem:** Services not running or wrong URLs

**Solution:**
```powershell
# Check if services are running
docker ps

# Or start them
.\Start-AllServices.ps1 -Docker
```

### "Swagger endpoint not found" (IdentityService)

**Problem:** IdentityService Swagger not configured

**Solution:** Ensure IdentityService [Program.cs](../../../IdentityService/src/IdentityService.API/Program.cs) has:
```csharp
builder.Services.AddFastEndpoints()
    .SwaggerDocument(o => { ... });

app.UseSwaggerGen();
```

### "IDENTITY_API_URL not defined"

**Problem:** Environment config missing

**Solution:** Update [environment.ts](src/config/environment.ts):
```typescript
const ENV = {
  dev: {
    API_URL: 'https://localhost:5006',          // OnlineMenuSaaS
    IDENTITY_API_URL: 'http://localhost:5002',  // IdentityService (NEW)
    // ... other config
  }
};
```

### Hooks Not Found

**Problem:** Old import paths

**Solution:**
❌ Old: `import { useLogin } from '@/server/autoGeneratedHooks/auth';`
✅ New: `import { useLogin } from '@/server/autoGeneratedHooks/identity/auth';`

## 🎓 Best Practices

### 1. Regenerate After API Changes
Whenever backend endpoints change:
```powershell
npm run generate:hooks
```

### 2. Use TypeScript Types
All generated hooks include TypeScript types:
```typescript
import type { TenantDto } from '@/server/autoGeneratedHooks/identity/models';

const tenant: TenantDto = {
  tenantId: '...',
  name: 'My Tenant',
  // TypeScript will enforce correct structure
};
```

### 3. Centralize API Calls
Create custom hooks that wrap generated ones:
```typescript
// hooks/useAuth.ts
import { useLogin as useLoginGenerated } from '@/server/autoGeneratedHooks/identity/auth';

export function useAuth() {
  const { mutate: login, ...rest } = useLoginGenerated();

  const handleLogin = (username: string, password: string) => {
    login({
      data: {
        method: 0,
        username,
        password,
      }
    });
  };

  return { handleLogin, ...rest };
}
```

### 4. Handle Base URLs Properly
Update `httpService.ts` to support multiple base URLs:
```typescript
const baseUrl = endpoint.startsWith('/api/auth') || endpoint.startsWith('/api/users')
  ? env.IDENTITY_API_URL
  : env.API_URL;
```

## 🆕 Adding a New Service (Step-by-Step Guide)

When you need to add a new backend service to the Orval code generation, follow these steps:

### Step 1: Add Environment Configuration

Edit `src/config/environment.ts` to add the API URL for all environments:

```typescript
// src/config/environment.ts
const ENV = {
  dev: {
    // ... existing config
    NEW_SERVICE_API_URL: 'http://localhost:PORT',  // Add your service URL
  },
  test: {
    // ... existing config
    NEW_SERVICE_API_URL: 'https://newservice-api.yourdomain.com',
  },
  prod: {
    // ... existing config
    NEW_SERVICE_API_URL: 'https://newservice-api.yourdomain.com',
  },
};
```

### Step 2: Create the HTTP Client

Create a new file `src/server/httpClientNewService.ts`:

```typescript
// src/server/httpClientNewService.ts
import { createHttpClient, type OrvalRequest, type OrvalMutator } from './createHttpClient';
import env from '../config/environment';
import { isNotEmptyString } from '../shared/utils/validators';

export type { OrvalRequest, OrvalMutator };

const DEFAULT_NEW_SERVICE_API_URL = 'http://localhost:PORT';

/**
 * HTTP client instance for NewService.
 */
export const newServiceInstance = createHttpClient({
  baseURL: isNotEmptyString(env.NEW_SERVICE_API_URL) ? env.NEW_SERVICE_API_URL : DEFAULT_NEW_SERVICE_API_URL,
  withToken: true,
  withCredentials: true,
});

export default newServiceInstance;
```

### Step 3: Create the Mutator

Create a new file `src/server/mutators/newServiceMutator.ts`:

```typescript
// src/server/mutators/newServiceMutator.ts
/**
 * NewService API mutator for Orval code generation.
 *
 * This file has no Expo dependencies at the module level, allowing Orval
 * to load it in Node.js during code generation.
 */
import { getMutator, type OrvalRequest, type OrvalMutator } from './registry';

export type { OrvalRequest, OrvalMutator };

/**
 * HTTP client for NewService API.
 */
export const newServiceInstance: OrvalMutator = async <
  TResp = unknown,
  TReq = unknown,
  TQry = unknown,
>(
  opts: OrvalRequest<TReq, TQry>,
): Promise<TResp> => {
  const realMutator = getMutator('newServiceInstance');
  return realMutator<TResp, TReq, TQry>(opts);
};

export default newServiceInstance;
```

### Step 4: Update the Registry

Edit `src/server/mutators/registry.ts`:

```typescript
// Add to the MutatorRegistry interface
interface MutatorRegistry {
  customInstance?: OrvalMutator;
  identityInstance?: OrvalMutator;
  questionerInstance?: OrvalMutator;
  contentInstance?: OrvalMutator;
  notificationInstance?: OrvalMutator;
  newServiceInstance?: OrvalMutator;  // Add this line
}

// Add to the registerMutators function
export function registerMutators(mutators: Partial<MutatorRegistry>): void {
  // ... existing lines
  if (mutators.newServiceInstance) registry.newServiceInstance = mutators.newServiceInstance;
}
```

### Step 5: Update orval.config.js

Add the new service configuration to `orval.config.js`:

```javascript
// orval.config.js
const NEW_SERVICE_MUTATOR = './src/server/mutators/newServiceMutator.ts';

module.exports = defineConfig({
  // ... existing configs

  // NewService API
  newServiceApi: {
    input: {
      target: './src/server/swagger/newservice.json',
      validation: false,
    },
    output: {
      target: './src/server/autoGeneratedHooks/newservice/index.ts',
      client: 'react-query',
      mode: 'tags-split',
      clean: true,
      prettier: true,
      schemas: './src/server/autoGeneratedHooks/newservice/models',
      override: {
        mutator: {
          path: NEW_SERVICE_MUTATOR,
          name: 'newServiceInstance',
        },
      },
    },
    hooks: {
      afterAllFilesWrite: [
        { command: 'npx prettier --write ./src/server/autoGeneratedHooks/newservice' },
        { command: 'echo "✅ Orval: NewService hooks generated"' },
      ],
    },
  },
});
```

### Step 6: Update Generate-ApiHooks.ps1

Edit `scripts/Generate-ApiHooks.ps1` to download the new service's swagger spec:

```powershell
# Add variable at the top
$NewServiceUrl = "http://localhost:PORT"

# Add parameter parsing
"-NewServiceUrl" {
    if ($i + 1 -lt $args.Count) {
        $NewServiceUrl = $args[$i + 1]
        $i++
    }
    continue
}

# Add download section
Write-Info "  [X/X] NewService API ($NewServiceUrl)..."
try {
    Save-OpenApi "$NewServiceUrl/swagger/v1/swagger.json" (Join-Path $SwaggerDir "newservice.json")
    Write-Success "    OK - NewService spec downloaded"
} catch {
    Write-Error "    FAIL - Could not download NewService spec"
    Write-Warning "    Make sure NewService is running on $NewServiceUrl"
}

# Add file verification
$newServiceFile = Join-Path $SwaggerDir "newservice.json"
if (-not (Test-Path $newServiceFile)) {
    Write-Error "NewService swagger file not found: $newServiceFile"
    exit 1
}
```

### Step 7: Register the Mutator at App Startup

Update your app's entry point (e.g., `App.tsx` or `_layout.tsx`) to register the new mutator:

```typescript
import { registerMutators } from './server/mutators/registry';
import { customInstance } from './server/httpClient';
import { identityInstance } from './server/httpClientIdentity';
import { questionerInstance } from './server/httpClientQuestioner';
import { contentInstance } from './server/httpClientContent';
import { notificationInstance } from './server/httpClientNotification';
import { newServiceInstance } from './server/httpClientNewService';  // Add import

registerMutators({
  customInstance,
  identityInstance,
  questionerInstance,
  contentInstance,
  notificationInstance,
  newServiceInstance,  // Add registration
});
```

### Step 8: Generate and Use

```powershell
# Generate hooks
npm run generate:hooks

# Use in components
import { useYourHook } from '@/server/autoGeneratedHooks/newservice';
```

### Checklist for Adding a New Service

- [ ] Mutator file created (`newServiceMutator.ts`)
- [ ] Registry updated with new instance type
- [ ] `orval.config.js` updated with new API config
- [ ] `Generate-ApiHooks.ps1` updated with new service download
- [ ] Mutator registered at app startup
- [ ] Hooks generated successfully
- [ ] Backend service is running with Swagger enabled
